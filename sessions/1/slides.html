Tests unitaires et TDD

Tests unitaires

C'est quoi ?

Un processus qui vérifie qu'une unité individuelle de code fait ce qu'elle est sensée faire

Tests unitaires (2)

Ses caractéristiques

  • est entièrement automatisé
    On ne s'intéresse pas aux tests manuels ici
  • est déterministe
    Pas d'aléatoire ou de magie, 2 lancements = 2 fois le même résultat
  • est isolé
    il s'exécute en mémoire et n'utilise pas d'i/o (fichiers, réseau, BDD, IPC, ...)
  • contrôle ce qu'il teste
    il isole le code testé par des mocks/stubs (bouchons)
  • est digne de confiance
    pas de faux positif
  • est rapide
    un test lent est une contrainte et perd son intérêt

Tests unitaires (3)

En bonus

  • less is more
    Petit et léger vaut mieux que gros et lourd
  • il est préférable qu'il soit lisible et maintenable
    Mais c'est vrai pour tout le code, non ?
  • il peut servir de documentation ou d'exemple
    Puisqu'il est une application directe du code
  • il file +2 intel et +4 esprit à son utilisateur
